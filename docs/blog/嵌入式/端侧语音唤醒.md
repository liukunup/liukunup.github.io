## 引言

"Hey Siri"、"OK Google"、"小爱同学"——说出唤醒词，设备即刻响应。这个看似简单的功能，背后是在**极低功耗、有限算力**下持续运行的实时音频处理系统。

本文科普端侧（On-Device / Edge）语音唤醒的核心技术原理与工程实现。

---

## 1. 什么是语音唤醒

语音唤醒（Keyword Spotting, KWS）指的是设备在**本地**持续监听音频流，实时检测特定关键词（唤醒词）的过程。

### 关键约束

| 约束 | 说明 |
|------|------|
| **Always-on** | 设备需 7×24 监听，不能关机 |
| **低功耗** | 电池供电设备下，唤醒模块功耗需控制在 mW 级 |
| **低延迟** | 唤醒反馈通常要求 < 500ms |
| **高唤醒率** | 安静环境 > 95%，嘈杂环境 > 90% |
| **低误唤醒** | 通常要求 < 1 次 / 24h |
| **离线本地** | 不依赖云端，保护隐私 |

### 整体流程

```
音频输入 → VAD(语音活动检测) → 特征提取 → 模型推理 → 后处理 → 唤醒决策
                    ↑                             ↓
              静音则丢弃                     非唤醒词则丢弃
```

## 2. 音频前端处理

### 2.1 麦克风与采样

- 常用采样率：**16kHz**（人类语音带宽 8kHz，满足奈奎斯特定理）
- 位深：16-bit PCM
- 声道：单声道

### 2.2 VAD — 语音活动检测

VAD 决定"是否有人说话"，避免无意义音频进入模型推理，降低功耗。

- **传统方法**：基于能量阈值、过零率、频谱平坦度
- **深度学习方法**：Silero VAD、WebRTC VAD

```python
# Silero VAD 示例
import torch
import torchaudio

model, utils = torch.hub.load(repo_or_dir="snakers4/silero-vad", model="silero_vad")
(get_speech_timestamps, _, _, _, _) = utils

wav = torchaudio.load("audio.wav")
timestamps = get_speech_timestamps(wav, model)
# 返回语音片段的起止时间
```

### 2.3 特征提取 — MFCC

原始音频波形维度高、信息冗余，需要提取更紧凑的声学特征。最常用的是 **MFCC (Mel-Frequency Cepstral Coefficients)**。

```
原始音频 (16kHz)
    ↓ 预加重 (Pre-emphasis)
    ↓ 分帧 (20-30ms 窗口) + 加窗 (Hamming)
    ↓ 时域 → 频域 (FFT)
    ↓ Mel 滤波器组 (40-80 个滤波器)
    ↓ Log
    ↓ DCT
    ↓ MFCC 特征 (通常 10-40 维)
```

```python
import librosa
import numpy as np

audio, sr = librosa.load("audio.wav", sr=16000)
mfcc = librosa.feature.mfcc(
    y=audio, sr=sr, n_mfcc=40,
    n_fft=512, hop_length=160,   # 10ms stride
    win_length=320                # 20ms window
)
# 输出: (40, T) 每帧 40 维 MFCC，步长 10ms
```

> 输入模型的特征通常是**上下文窗口**：将连续 N 帧 MFCC 拼接，形成 `1 × Freq × Time` 的二维图像。例如 40 维 MFCC × 100 帧（约 1 秒语音）。

## 3. 唤醒模型架构

### 3.1 模型演进

```
传统方法                       深度学习方法
┌─────────────┐             ┌──────────────────┐
│ DTW 模板匹配 │             │  DNN / CNN / RNN  │
│ (2000s)     │     ──→     │  (2015+)          │
│ GMM-HMM     │             │  DS-CNN / CRNN    │
│ 精度低，泛化差│             │  Transformer / Mamba │
└─────────────┘             │  精度高，参数少     │
                            └──────────────────┘
```

### 3.2 轻量级模型选型

| 模型 | 参数量 | 精度 | 特点 |
|------|--------|------|------|
| DS-CNN (Depthwise Separable CNN) | ~100K | ⭐⭐⭐ | Google 提出，MCU 首选 |
| CRNN (CNN + GRU/LSTM) | ~200K | ⭐⭐⭐⭐ | 时序建模好，稍重 |
| FSMN (Feedforward Sequential Memory) | ~200K | ⭐⭐⭐⭐ | 流式友好，低延迟 |
| DTCN (Dilated Timewise CNN) | ~100K | ⭐⭐⭐⭐ | 大感受野，参数量少 |
| Transformer (轻量版) | ~500K | ⭐⭐⭐⭐⭐ | 效果好，资源要求高 |
| EdgeSpeechNet | ~50K | ⭐⭐⭐ | 极端轻量，适合 MCU |

### 3.3 DS-CNN 结构

```
Input: 1 × 40 × 100 (MFCC)
  ↓ Conv2D 3×3, 64 (Depthwise)
  ↓ Conv2D 1×1, 64 (Pointwise)
  ↓ Conv2D 3×3, 64 (Depthwise)
  ↓ Conv2D 1×1, 64 (Pointwise)
  ↓ ...
  ↓ Global Average Pooling
  ↓ Dense, num_classes
  ↓ Softmax
Output: 各唤醒词的概率
```

```python
# TensorFlow DS-CNN 示例
import tensorflow as tf

def dscnn_model(input_shape=(40, 100, 1), num_classes=2):
    inputs = tf.keras.Input(shape=input_shape)

    x = tf.keras.layers.DepthwiseConv2D((3, 3), depth_multiplier=1, padding='same')(inputs)
    x = tf.keras.layers.BatchNormalization()(x)
    x = tf.keras.layers.ReLU()(x)
    x = tf.keras.layers.Conv2D(64, (1, 1), padding='same')(x)
    x = tf.keras.layers.BatchNormalization()(x)
    x = tf.keras.layers.ReLU()(x)

    x = tf.keras.layers.DepthwiseConv2D((3, 3), strides=2, padding='same')(x)
    x = tf.keras.layers.Conv2D(128, (1, 1), padding='same')(x)
    x = tf.keras.layers.BatchNormalization()(x)
    x = tf.keras.layers.ReLU()(x)

    x = tf.keras.layers.GlobalAveragePooling2D()(x)
    x = tf.keras.layers.Dropout(0.3)(x)
    outputs = tf.keras.layers.Dense(num_classes, activation='softmax')(x)

    return tf.keras.Model(inputs, outputs)

model = dscnn_model()
model.summary()  # ~95K params
```

### 3.4 CRNN 结构

```
Input: 40 × 100 (MFCC 帧)
  ↓ Conv2D × N (提取局部频谱模式)
  ↓ Permute (转为时序特征)
  ↓ GRU / LSTM × N (时序建模)
  ↓ Dense
  ↓ Softmax
```

## 4. 损失函数与训练策略

### 4.1 Max-Pooling 损失

将模型在一段音频上的各帧输出做 max pooling，取最大值作为整段音频的得分：

```python
def max_pooling_loss(logits, target, margin=0.2):
    """
    logits: (B, T, C) 每帧输出
    target: (B,) 标签
    """
    # 对时间维度取最大值
    max_logits = torch.max(logits, dim=1)  # (B, C)
    # 交叉熵
    loss = F.cross_entropy(max_logits, target)
    return loss
```

### 4.2 CTC 损失

CTC 在训练时不需要对齐帧与标签，适合唤醒词这种"长音频中只有一小段是关键词"的场景。

```python
import torch.nn.functional as F

def ctc_loss(logits, targets, input_lengths, target_lengths):
    # logits: (T, B, C)  T=帧数
    log_probs = F.log_softmax(logits, dim=-1)
    loss = F.ctc_loss(log_probs, targets, input_lengths, target_lengths, blank=0)
    return loss
```

### 4.3 Kullback-Leibler Divergence + 负样本挖掘

用 KL 散度让正样本的预测分布集中在目标标签附近，同时对负样本（非唤醒词）进行**难例挖掘**。

### 数据集

| 数据集 | 描述 | 规模 |
|--------|------|------|
| Google Speech Commands | 35 个常见英文词 | 105,000 条 |
| 自定义唤醒词采集 | 目标唤醒词 + 混淆词 + 噪音 | 千人千时 |
| 远场噪音数据 | 电视、厨房、街道等噪音 | — |

## 5. 推理与流式处理

### 5.1 流式推理架构

唤醒系统不能等用户说完一整段话才判断，必须**逐帧（或逐块）流式推理**。

```
16kHz PCM 流
  ↓ 环形缓冲区 (约 1-2 秒)
  ↓ 每 10ms / 20ms 滑窗
  ↓ 特征提取 (增量计算)
  ↓ 模型推理
  ↓ 后处理 + 平滑
  ↓ 得分 > 阈值 ? → 唤醒
```

### 5.2 平滑策略

单帧得分波动大，通常需要平滑后决策：

```python
class WakeWordDetector:
    def __init__(self, threshold=0.7, smooth_window=10):
        self.threshold = threshold
        self.scores = []  # 滑动窗口
        self.smooth_window = smooth_window

    def process_frame(self, score):
        self.scores.append(score)
        if len(self.scores) > self.smooth_window:
            self.scores.pop(0)

        smoothed = sum(self.scores) / len(self.scores)
        if smoothed > self.threshold:
            self.scores = []
            return True  # 唤醒!
        return False
```

### 5.3 两级唤醒架构

```
第一级 (低功耗 DSP)
  ↓ 简单模型 (约 10K 参数)
  ↓ 低精度 (可能会误触发)
  ↓

第二级 (主 CPU / NPU)
  ↓ 复杂模型 (约 200K 参数)
  ↓ 高精度 (确认后最终唤醒)
  ↓

系统唤醒
```

第一级 DSP 功耗极低（< 1mW），持续运行；只有第一级触发后才唤醒主处理器执行第二级验证。这是目前手机和智能音箱的主流方案。

## 6. 模型部署与量化

### 6.1 部署流程

```
PyTorch / TensorFlow 训练
    ↓
ONNX 导出
    ↓
模型量化 (FP32 → INT8)
    ↓
目标平台部署
  ├── TFLite Micro (MCU)
  ├── ONNX Runtime (Linux/Android)
  ├── NCNN / MNN (移动端)
  ├── SNPE (Qualcomm DSP)
  └── CoreML (Apple)
```

### 6.2 量化

```python
# TensorFlow 量化示例
converter = tf.lite.TFLiteConverter.from_keras_model(model)
converter.optimizations = [tf.lite.Optimize.DEFAULT]
converter.representative_dataset = representative_dataset
converter.target_spec.supported_types = [tf.float16]  # 或 tf.int8
tflite_quant_model = converter.convert()
```

量化效果：

| 精度 | 模型大小 | 推理速度 | 精度损失 |
|------|---------|---------|---------|
| FP32 | 100% | 1x | 基准 |
| FP16 | 50% | 1.5-2x | ~0% |
| INT8 | 25% | 3-4x | < 1% |

### 6.3 端侧推理代码

```cpp
// TFLite Micro 推理示例 (C++)
#include "tensorflow/lite/micro/all_ops_resolver.h"
#include "tensorflow/lite/micro/micro_interpreter.h"
#include "tensorflow/lite/schema/schema_generated.h"

const unsigned char* model_data; // 量化后的模型
static tflite::MicroMutableOpResolver<10> resolver;
static tflite::MicroInterpreter static_interpreter(
    model_data, resolver, tensor_arena, kTensorArenaSize);

void run_inference(float* mfcc_input, float* output_score) {
    // 拷贝输入
    memcpy(interpreter->input(0)->data.f, mfcc_input,
           interpreter->input(0)->bytes);

    // 推理
    interpreter->Invoke();

    // 获取输出
    output_score[0] = interpreter->output(0)->data.f[1]; // 唤醒词概率
}
```

## 7. 评估指标

### 7.1 唤醒评估

| 指标 | 含义 | 测试方式 |
|------|------|---------|
| 唤醒率 (Wake-up Rate) | 说唤醒词后被唤醒的比例 | 不同 SNR、距离下多次测试 |
| 误唤醒率 (False Alarm Rate) | 非唤醒词/噪音下被唤醒的次数 | 长时间背景噪声 + 混淆词 |
| 响应延迟 (Latency) | 从说完唤醒词到系统响应 | 音频打标测量 |
| ROC 曲线 | TPR vs FPR 随阈值变化 | 遍历阈值绘制 |
| DET 曲线 | 漏唤醒率 vs 误唤醒率 | 同 ROC 的另一种视角 |

### 7.2 参考指标

| 场景 | 唤醒率 | 误唤醒率 | 延迟 |
|------|--------|---------|------|
| 手机 (Hey Siri) | > 95% (安静) | < 1 次/天 | < 500ms |
| 智能音箱 (Alexa) | > 90% (3米远场) | < 1 次/天 | < 500ms |
| 耳机 (低功耗) | > 90% | < 2 次/天 | < 300ms |

## 9. 端侧挑战与趋势

| 挑战 | 说明 | 趋势 |
|------|------|------|
| 功耗 | Always-on 必须在 mW 级 | 专用 DSP / NPU 分离唤醒 |
| 精度 | 远场噪音下保持高唤醒率 | 自监督 + 数据增强 |
| 自定义唤醒词 | 用户可自由设置 | 少样本微调 / 零样本学习 |
| 多语言 | 中文、方言等 | 跨语言预训练模型 |
| 隐私 | 数据不出设备 | 联邦学习优化模型 |
| 抗噪 | 多人对话、电视背景音 | 定向麦克风 + 波束成形 |
