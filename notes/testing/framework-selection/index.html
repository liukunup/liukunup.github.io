C++: GoogleTest

GoogleTest 是 C++ 领域事实标准的测试框架，搭配 GoogleMock 提供完整的 Mock 能力。

核心用法

#include <gtest/gtest.h>
#include <gmock/gmock.h>

// 被测函数
int Add(int a, int b) { return a + b; }

// 测试用例
TEST(MathTest, AddPositive) {
    EXPECT_EQ(Add(1, 2), 3);
    ASSERT_GT(Add(0, 1), 0);
}

TEST(MathTest, AddNegative) {
    EXPECT_EQ(Add(-1, -2), -3);
}

// 测试固件
class MyTestFixture : public ::testing::Test {
protected:
    void SetUp() override { /* 测试前准备 */ }
    void TearDown() override { /* 测试后清理 */ }
};

TEST_F(MyTestFixture, DoSomething) {
    // ...
}

// 参数化测试
class ParamTest : public ::testing::TestWithParam<std::tuple<int, int, int>> {};
TEST_P(ParamTest, Add) {
    auto [a, b, expected] = GetParam();
    EXPECT_EQ(Add(a, b), expected);
}
INSTANTIATE_TEST_SUITE_P(Default, ParamTest,
    ::testing::Values(
        std::make_tuple(1, 2, 3),
        std::make_tuple(-1, 1, 0)
    ));

// Mock 对象
class DatabaseMock : public Database {
public:
    MOCK_METHOD(std::string, Query, (int id), (override));
};

TEST(DbTest, QueryUser) {
    DatabaseMock mock;
    EXPECT_CALL(mock, Query(42))
        .Times(1)
        .WillOnce(::testing::Return("Alice"));

    UserService svc(&mock);
    auto name = svc.GetUserName(42);
    EXPECT_EQ(name, "Alice");
}

选择建议

场景框架理由
大型工程GoogleTest功能全、Mock 内置、社区成熟
头文件测试Catch2单头文件集成、BDD 风格支持
资源敏感设备doctest极小编译开销和二进制体积

Python: pytest

pytest 是 Python 社区最流行的测试框架，以简洁的 assert 和强大的 fixture 系统著称。

核心用法

# test_math.py
import pytest


# 基本测试
def test_add():
    assert add(1, 2) == 3
    assert add(-1, 1) == 0


# 参数化测试
@pytest.mark.parametrize("a,b,expected", [
    (1, 2, 3),
    (-1, 1, 0),
    (0, 0, 0),
])
def test_add_parametrize(a, b, expected):
    assert add(a, b) == expected


# Fixture
@pytest.fixture
def db_connection():
    conn = create_connection()
    yield conn
    conn.close()


def test_query(db_connection):
    result = db_connection.query("SELECT 1")
    assert result is not None


# conftest.py — 共享 fixture
# 将 fixture 定义在 conftest.py 中，同目录下所有测试自动可用


# Mock
def test_get_user(mocker):
    mocker.patch("module.get_user_from_db", return_value={"name": "Alice"})
    result = get_user(42)
    assert result["name"] == "Alice"


# 异常测试
def test_raises():
    with pytest.raises(ValueError, match="invalid"):
        func_that_raises()


# skip / xfail
@pytest.mark.skip(reason="not implemented yet")
def test_future():
    pass

@pytest.mark.xfail
def test_known_bug():
    assert 1 == 2

pytest vs unittest

对比项pytestunittest
断言语法assertself.assertEqual / assertTrue
参数化@pytest.mark.parametrize@unittest.parameterized 或子测试
Fixture@pytest.fixture（模块级）setUp / tearDown 方法
插件生态丰富（pytest-cov, xdist, etc.）有限
学习成本低低
推荐度⭐ 首选⭐ 栈底

新项目首选 pytest。unittest 是标准库内置方案，无需额外依赖，适合简单场景或限制依赖的场合。

Java: JUnit 5

JUnit 5 是现代 Java 的事实标准测试框架，由 Platform + Jupiter + Vintage 三层架构组成。

核心用法

import org.junit.jupiter.api.*;
import static org.junit.jupiter.api.Assertions.*;

class CalculatorTest {

    @BeforeAll
    static void setupAll() { /* 类级别初始化 */ }

    @BeforeEach
    void setup() { /* 每个测试前执行 */ }

    @AfterEach
    void teardown() { /* 每个测试后执行 */ }

    @AfterAll
    static void teardownAll() { /* 类级别清理 */ }

    @Test
    void testAdd() {
        assertEquals(3, calculator.add(1, 2));
        assertNotEquals(5, calculator.add(1, 2));
    }

    @Test
    void testThrows() {
        assertThrows(IllegalArgumentException.class,
            () -> calculator.divide(1, 0));
    }

    // 参数化测试
    @ParameterizedTest
    @CsvSource({"1,2,3", "-1,1,0"})
    void testAddParam(int a, int b, int expected) {
        assertEquals(expected, calculator.add(a, b));
    }

    // 动态测试
    @TestFactory
    Collection<DynamicTest> dynamicTests() {
        return List.of(
            DynamicTest.dynamicTest("add 1+2", () -> assertEquals(3, calculator.add(1, 2)))
        );
    }
}

Mockito

import static org.mockito.Mockito.*;

@ExtendWith(MockitoExtension.class)
class UserServiceTest {

    @Mock
    UserRepository userRepository;

    @InjectMocks
    UserService userService;

    @Test
    void testGetUserName() {
        when(userRepository.findById(42))
            .thenReturn(new User("Alice"));

        String name = userService.getUserName(42);
        assertEquals("Alice", name);
        verify(userRepository).findById(42);
    }
}

JUnit 5 vs TestNG

对比项JUnit 5TestNG
架构Platform + Jupiter + Vintage单一
注解风格@Test / @BeforeEach / @ParameterizedTest@Test / @BeforeMethod / @DataProvider
参数化@CsvSource / @ValueSource / @MethodSource@DataProvider
依赖注入✅ 参数解析器✅ Guice
并行执行✅ 内置✅ 内置
社区趋势⭐ 主流存量项目

新项目首选 JUnit 5。TestNG 曾经在参数化和并行方面领先，但 JUnit 5 已经全面追上并超越。

Go: testing + testify

Go 标准库自带 testing 包，配合 testify 可以大幅提升测试编写体验。

核心用法

// math_test.go
package main

import (
    "testing"
    "github.com/stretchr/testify/assert"
    "github.com/stretchr/testify/require"
    "github.com/stretchr/testify/suite"
)

// 标准 testing 方式
func TestAdd(t *testing.T) {
    result := Add(1, 2)
    if result != 3 {
        t.Errorf("expected 3, got %d", result)
    }
}

// testify 断言
func TestAddWithAssert(t *testing.T) {
    assert.Equal(t, 3, Add(1, 2))
    assert.NotEqual(t, 0, Add(1, 2))
}

// testify require（失败时立即终止）
func TestDivide(t *testing.T) {
    result, err := Divide(10, 0)
    require.Error(t, err)    // 没有 err 时直接终止
    assert.Equal(t, 0, result)
}

// Table-driven 测试
func TestAddTable(t *testing.T) {
    tests := []struct {
        name string
        a, b int
        expected int
    }{
        {"positive", 1, 2, 3},
        {"negative", -1, 1, 0},
        {"zero", 0, 0, 0},
    }
    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            assert.Equal(t, tt.expected, Add(tt.a, tt.b))
        })
    }
}

// Suite
type MySuite struct {
    suite.Suite
    db *DB
}

func (s *MySuite) SetupSuite() {
    s.db = NewDB()
}

func (s *MySuite) TearDownSuite() {
    s.db.Close()
}

func (s *MySuite) TestQuery() {
    result := s.db.Query(42)
    s.Equal("Alice", result.Name)
}

func TestRunSuite(t *testing.T) {
    suite.Run(t, new(MySuite))
}

Go Mock

// 方式一：testify/mock
type DB struct {
    mock.Mock
}

func (m *DB) GetUser(id int) (string, error) {
    args := m.Called(id)
    return args.String(0), args.Error(1)
}

func TestGetUserName(t *testing.T) {
    db := new(DB)
    db.On("GetUser", 42).Return("Alice", nil)

    svc := NewUserService(db)
    name, err := svc.GetUserName(42)
    assert.NoError(t, err)
    assert.Equal(t, "Alice", name)
    db.AssertExpectations(t)
}

选型决策要点

  1. Mock 需求：C++ 选 GoogleMock（内置），Java 选 Mockito，Python 选 pytest-mock，Go 选 testify/mock 或 gomock
  2. 参数化测试：所有框架均支持，Go 的 table-driven 是其特色
  3. Fixture 管理：pytest 的 fixture 体系最具表达力，JUnit 5 次之
  4. 并行执行：pytest 需 xdist 插件，JUnit 5 / Go 原生支持
  5. 覆盖率：各自有工具链，Go 的覆盖率工具链最简单（内置 -cover）
  6. IDE 集成：Java + IntelliJ 体验最好，Go + GoLand / VSCode 次之
