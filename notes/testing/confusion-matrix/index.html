混淆矩阵及其衍生指标

约 1514 字大约 5 分钟

2026-06-16

混淆矩阵

混淆矩阵（Confusion Matrix）是分类模型最基本的评价工具，以矩阵形式呈现预测结果与真实标签的对照关系。

二分类混淆矩阵

预测为正 (Positive)预测为负 (Negative)
实际为正TP (True Positive)FN (False Negative)
实际为负FP (False Positive)TN (True Negative)

四个基本量：

符号名称含义直观理解
TP真正例预测为正，实际也为正猜对了（正样本）
TN真负例预测为负，实际也为负猜对了（负样本）
FP假正例预测为正，实际为负误报（虚惊一场）
FN假负例预测为负，实际为正漏报（错过了）

二分类衍生指标

准确率 (Accuracy)

所有预测中，预测正确的比例。

Accuracy=TP+TNTP+TN+FP+FN\text{Accuracy} = \frac{TP + TN}{TP + TN + FP + FN} Accuracy=TP+TN+FP+FNTP+TN​

适用：正负样本均衡时。样本不平衡时准确率可能虚高（如 99% 负样本，全判负也有 99%）。

精确率 (Precision)

在所有被预测为正的样本中，实际为正的比例。

Precision=TPTP+FP\text{Precision} = \frac{TP}{TP + FP} Precision=TP+FPTP​

关注：误报的控制。值越高，说明误报越少。

召回率 (Recall / Sensitivity)

在所有实际为正的样本中，被正确预测为正的比例。

Recall=TPTP+FN\text{Recall} = \frac{TP}{TP + FN} Recall=TP+FNTP​

关注：漏报的控制。值越高，说明漏报越少。

特异度 (Specificity)

在所有实际为负的样本中，被正确预测为负的比例。

Specificity=TNTN+FP\text{Specificity} = \frac{TN}{TN + FP} Specificity=TN+FPTN​

F1 Score

精确率和召回率的调和平均，综合衡量两者。

F1=2×Precision×RecallPrecision+RecallF_1 = 2 \times \frac{\text{Precision} \times \text{Recall}}{\text{Precision} + \text{Recall}} F1​=2×Precision+RecallPrecision×Recall​

适用：当精确率和召回率需要平衡时。F1 对两者中的低值更敏感。

指标对比速览

指标公式对什么敏感需高需低
Accuracy(TP+TN) / (TP+TN+FP+FN)整体正确率越高越好
PrecisionTP / (TP+FP)FP（误报）越高越好
RecallTP / (TP+FN)FN（漏报）越高越好
SpecificityTN / (TN+FP)FP（误报）越高越好
F12 × P × R / (P + R)Precision 和 Recall 的平衡越高越好

场景示例

不同场景对指标的关注点不同：

场景优先指标原因
医疗诊断（癌症筛查）Recall宁可误报也不能漏诊
垃圾邮件过滤Precision宁可有垃圾邮件进入收件箱，也不能误判正常邮件
推荐系统Precision + Recall平衡准确覆盖和精准推荐
搜索引擎Recall尽可能多的返回相关结果

多分类混淆矩阵

多分类时，混淆矩阵是一个 N × N 矩阵，每个元素 CijC_{ij}Cij​ 表示实际为类别 iii 但被预测为类别 jjj 的样本数。

        预测类别
        0   1   2
实际 0 [10,  2,  1]   ← 类别0: 10个正确, 2个被误判为1, 1个被误判为2
    1 [ 3,  8,  1]   ← 类别1: 8个正确, 3个被误判为0, 1个被误判为2
    2 [ 0,  1,  9]   ← 类别2: 9个正确, 1个被误判为1

多分类指标通常将每个类别视为"正类"，其余视为"负类"，然后计算宏平均或加权平均。

Macro Average（宏平均）

每个类别独立计算指标，然后取算术平均。

Macro Precision=1N∑i=1NPi\text{Macro Precision} = \frac{1}{N} \sum_{i=1}^{N} P_i Macro Precision=N1​i=1∑N​Pi​

特点：每个类别权重相同，不受样本量影响。对小类别更公平。

Weighted Average（加权平均）

每个类别的指标按样本量加权平均。

Weighted F1=∑i=1Nwi×F1i\text{Weighted F1} = \sum_{i=1}^{N} w_i \times F1_i Weighted F1=i=1∑N​wi​×F1i​

特点：考虑各类别样本量，大类别主导。

Micro Average（微平均）

将所有类别的 TP / FP / FN 汇总后统一计算指标。

Micro F1=∑TP∑TP+∑FP\text{Micro F1} = \frac{\sum TP}{\sum TP + \sum FP} Micro F1=∑TP+∑FP∑TP​

特点：等价于全局 Accuracy，受大类别主导。

PR 曲线

PR 曲线以 Recall 为横轴、Precision 为纵轴，展示不同阈值下的表现。

  • 曲线越靠近右上角，模型越好
  • AP (Average Precision)：PR 曲线下的面积，数值越大越好
  • mAP (mean AP)：多类别的 AP 取平均，目标检测中的核心指标

ROC 曲线与 AUC

ROC 曲线以 FPR (False Positive Rate) 为横轴、TPR (True Positive Rate，即 Recall) 为纵轴。

FPR=FPFP+TN=1−Specificity\text{FPR} = \frac{FP}{FP + TN} = 1 - \text{Specificity} FPR=FP+TNFP​=1−Specificity

AUC 的含义

  • AUC = 1.0：完美分类器
  • AUC = 0.5：随机猜测（无区分能力）
  • AUC < 0.5：比随机还差（可能标签反了）

PR 曲线 vs ROC 曲线

对比项PR 曲线ROC 曲线
横轴RecallFPR
对不平衡敏感✅ 敏感❌ 不敏感
适用场景稀有事件检测类别相对均衡
负样本增多时曲线下移基本不变

经验法则：正样本稀少时（如异常检测、罕见病诊断）用 PR 曲线；正负均衡时用 ROC 曲线。

多标签分类指标

当每个样本可能属于多个标签时（如给一篇文章打多个主题标签）：

Hamming Loss

预测错误的标签占总标签数的比例（越低越好）。

Hamming Loss=1N×L∑i=1N∑j=1L1(y^ij≠yij)\text{Hamming Loss} = \frac{1}{N \times L} \sum_{i=1}^{N} \sum_{j=1}^{L} \mathbb{1}(\hat{y}_{ij} \neq y_{ij}) Hamming Loss=N×L1​i=1∑N​j=1∑L​1(y^​ij​=yij​)

Exact Match Ratio (子集精度)

所有标签全部预测正确的样本比例（越严格越难高）。

Exact Match=1N∑i=1N1(y^i=yi)\text{Exact Match} = \frac{1}{N} \sum_{i=1}^{N} \mathbb{1}(\hat{y}_i = y_i) Exact Match=N1​i=1∑N​1(y^​i​=yi​)
